# 0010. 용어 도움말은 검수된 정적 콘텐츠 — LLM 실시간 생성 금지

- **날짜**: 2026-07-12
- **상태**: Accepted
- **관련**: SoT [PART E1](../SoT.md#e1-adr-새-저장소-docsadr에-재기록) #10, [A5.11](../SoT.md#a511-이해-지원--학습-투자-초보-전제), [A6.12](../SoT.md#a612-초보자-이해-지원--용어-도움말--학습-요소)

## 맥락

오너는 투자 초보자이다(A1). 시스템이 보여주는 팩터·지표·리스크 용어를 오너가 이해하지 못하면, 승인형
모드에서의 주문 후보 승인/거절도, 전략 설정도 형식적인 행위가 되어버린다 — 즉 이해 지원은 부가 기능이
아니라 **안전 기능**이다. 동시에 LLM이 매 요청마다 용어 설명을 실시간 생성하면, 확인되지 않은 오설명이
그대로 노출될 위험과 반복 호출 비용이 발생한다.

## 결정

용어 도움말은 **검수된 정적 콘텐츠만 사용**하고, **LLM 실시간 생성은 금지**한다. LLM으로 초안을 만들어
오너가 검수한 후 정적 콘텐츠에 반영하는 제작 워크플로는 허용한다.

## 결과

- 단일 소스는 저장소 내 `glossary.ko.yaml`이다 — git으로 버전 관리되며 변경은 코드 리뷰로 검수된다.
  DB 테이블이 아닌 읽기 전용 콘텐츠로 취급한다 (A6.12).
- 백엔드가 이 파일을 로드해 API로 프론트엔드에 제공하며, 알림·브리핑·이메일 문구 생성 시에도 **동일
  항목을 참조**해 화면과 이메일의 설명이 불일치하지 않도록 한다.
- 프론트엔드 공통 컴포넌트(`<Term>`)가 탭/클릭 팝오버로 정의·해석·주의점을 노출하며, 전문용어를
  노출하는 모든 새 화면은 해당 용어의 용어집 연결까지가 완료 기준(DoD)이다.
- 근거: **틀린 설명은 설명이 없는 것보다 해롭고**, 정적 콘텐츠는 검수 가능하며 호출 비용이 0이다.
  반면 LLM 설명 생성(A6.1 단계 5)은 점수 변동 해석이라는 별개 용도로만 사용되며, 원칙 7에 따라 여전히
  점수 계산에는 관여하지 않는다.
