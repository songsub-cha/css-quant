# 0007. 2-레짐으로 시작, 4-레짐은 튜닝 이후

- **날짜**: 2026-07-11
- **상태**: Accepted
- **관련**: SoT [PART E1](../SoT.md#e1-adr-새-저장소-docsadr에-재기록) #7, [A6.2](../SoT.md#a62-시장-레짐-mvp-2레짐), [A6.10](../SoT.md#a610-점수-모델-진화-경로-비-ml)

## 맥락

AI 점수 파이프라인은 시장 상황에 따라 팩터 가중치를 다르게 적용해야 하지만(A6.1), 레짐을 얼마나
세분화할지는 데이터와 백테스트로 검증되지 않은 상태에서는 과설계 위험이 있다. 레짐 세분화가 지나치면
각 레짐의 표본이 부족해져 가중치 튜닝 자체가 불안정해진다.

## 결정

MVP는 **NORMAL / DEFENSIVE 2-레짐**으로 시작한다. 4-레짐(Risk-On/Neutral/High-Vol/Risk-Off) 세분화는
가중치 튜닝(A6.10 단계 1)이 완료된 이후 Phase 11에서 도입한다.

## 결과

- 레짐 판정은 결정론적 규칙으로 매 거래일 1회 수행한다: `KOSPI 종가 > 200일 MA AND (VKOSPI < 25 또는
  20일 변동성 < 1.5%)`이면 NORMAL, 아니면 DEFENSIVE (A6.2).
- DEFENSIVE → NORMAL 전환은 히스테리시스(연속 3거래일 조건 만족)를 적용해 안전 우선으로 설계하고,
  NORMAL → DEFENSIVE는 즉시 전환한다. 레짐 기록이 없는 콜드 스타트는 DEFENSIVE로 시작한다.
- 이 최소 복잡도는 **검증 가능성**을 우선한 선택이다 — 레짐 4분화, 섹터 중립화, 팩터 추가는 모두
  A6.10의 비-ML 진화 경로(상시 IC 모니터링 → 가중치 튜닝 → 팩터 추가 → 섹터 중립화 → 레짐 4분화)를
  따라 순차적으로만 도입한다.
