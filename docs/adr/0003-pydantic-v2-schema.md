# 0003. Pydantic v2를 스키마 정의 표준으로 채택

- **날짜**: 2026-07-11
- **상태**: Accepted
- **관련**: SoT [PART E1](../SoT.md#e1-adr-새-저장소-docsadr에-재기록) #3, [A6.5](../SoT.md#a65-전략-엔진), [B4.7](../SoT.md#b4-표준), [B5](../SoT.md#b5-검증된-구현-결정-구-저장소에서-확인된-함정--반드시-승계) 기술 스택

## 맥락

전략 config, 도메인 모델, API 요청/응답 스키마를 서버 측에서 검증해야 하며, 동시에 프론트엔드(Zod)와
스키마 정의를 동기화할 방법이 필요하다. FastAPI 생태계와의 정합성, 타입 검증 성능, OpenAPI 자동 생성
지원이 선택 기준이다.

## 결정

백엔드 스키마/도메인 모델 정의는 **Pydantic v2**를 표준으로 사용한다.

## 결과

- 전략 `config` 스키마는 Pydantic으로 정의되고, 이를 근거로 생성된 OpenAPI 스펙에서 프론트엔드 Zod
  스키마를 자동 동기화한다 (A6.5, B4.7 "생성 타입 동기화": 백엔드 OpenAPI → `packages/api-types` 자동
  생성, 수동 편집 금지).
- 전략 설정은 저장 전 **Zod + Pydantic 양쪽에서 검증**된다 (A5.3) — 프론트는 사용자 경험을 위한 즉시
  검증, 백엔드는 신뢰 경계에서의 최종 검증을 담당한다.
- FastAPI 자체가 Pydantic v2 기반이므로 요청/응답 검증과 도메인 스키마 정의가 하나의 도구로 통일된다.
