# 0009. 정보 모니터링(공시 감시·일일 브리핑) MVP 편입

- **날짜**: 2026-07-11
- **상태**: Accepted
- **관련**: SoT [PART E1](../SoT.md#e1-adr-새-저장소-docsadr에-재기록) #9, [A1](../SoT.md#a1-한-줄-정의), [A6.11](../SoT.md#a611-정보-모니터링--공시-감시--일일-브리핑)

## 맥락

제품 정의는 매수·매도 전략 실행이라는 한 축만이 아니라, 가격·재무·공시·시장 지표 등 투자 관련 정보를
매 거래일 자동 수집·모니터링해 오너의 투자 판단을 돕는다는 두 번째 축을 포함한다(A1). 이 축이 MVP에
포함되지 않으면 셀프호스팅 무인 운영에서 오너가 시장 상황을 파악할 유일한 수단이 대시보드를 직접
열어보는 것뿐이게 된다.

## 결정

**공시 감시**(DART 화이트리스트 기반 중요 공시 알림)와 **일일 브리핑**을 MVP(Phase 6)에 편입한다.
단, **비정형 정보(공시·뉴스)는 점수·신호·주문 생성에 자동 개입하지 않는다** — 표시·알림 전용으로
제한한다.

## 결과

- 공시 감시는 보유(모의 포함)·관심 종목을 대상으로 DART 공시목록 API를 일 2회(17:35, 익일 08:00)
  수집하고, 설정 테이블 기반 화이트리스트(유상증자, 감자, 합병·분할, 상장폐지 관련 등)에 매칭되면
  즉시 알림과 종목 상세 플래그를 발생시킨다 (A6.11).
- 일일 브리핑은 파이프라인 완료 후 17:45에 파이프라인 상태·레짐·포트폴리오 요약·신규 후보·점수
  급변·중요 공시·오늘의 용어를 묶어 1건으로 발송한다 (A6.11, D6).
- 이 제한은 원칙 7(숫자는 결정론적 코드, 해석은 LLM)과 백테스트 재현성을 지키기 위한 것이다 — 공시나
  뉴스처럼 비정형이고 소급 검증이 어려운 정보가 매매 로직에 섞이면 백테스트 결과를 실거래에서 재현할
  수 없게 된다.
- 공시·뉴스를 전략의 입력(이벤트 플래그)으로 활용하는 것은 Phase 12에서 별도로 검토한다.
